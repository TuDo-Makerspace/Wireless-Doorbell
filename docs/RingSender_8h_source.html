31 /**
32  * @brief The RingSender class.
33  *
34  * The RingSender class is used to send a ring message to all bells,
35  * as well as checking how many bells have responded.
36  *
37  * It does so by creating a RingTX instance for each bell, and
38  * checking if the bell has responded.
39  *
40  * @attention For convenience reasons, we do not parse a fixed
41  * list of bell IP addresses to the RingReceiver, but instead rely
42  * on bells to reserve ip addresses following the door's ip address.
43  *
44  * ```
45  * Ex.:
46  * Door IP: 192.168.0.30
47  * Bell1 IP: 192.168.0.31
48  * Bell2 IP: 192.168.0.32
49  * ...
50  * BellX IP: 192.168.0.30 + X
51  * ```
52  *
53  * To add bells to our network, we simply only have to update
54  * the doors N_BELLS configuration parameter in the config.h file.
55  */
